# regen-prd-writer

> 一个封装了产品经理核心思维的 skill，帮助用户高效写出结构化、可执行的产品需求文档（PRD）。支持传统应用、AI 增强应用、AI Agent 应用三类产品——自动识别类型并加载对应章节模板。无论是有产品想法的独立开发者，还是需要提效的产品经理，都能从中获益。

---

## 简介

`regen-prd-writer` 把产品经理的核心思维封装进了 skill，帮助用户高效地把想法变成结构化、可执行的需求文档。

适合场景：
- Vibe coding 场景中的独立开发者
- 想做产品但没有 PM 背景的创业者
- 需要提效或自查文档质量的产品经理
- 需要把模糊想法结构化的任何人
- 正在构建 AI Agent 产品、需要写 Agent 故事和提示词策略的开发者

---

## 核心功能

**三种工作模式：**

| 模式 | 触发条件 |
|------|---------|
| **模式 A：从零生成** | 有想法，还没有文档 |
| **模式 B：评估改进** | 已有文档，想检查或优化 |
| **模式 C：增量更新** | 已有文档，想追加新需求 |

**三级产品类型自动识别：**

| 产品类型 | 落地章节 |
|---------|---------|
| **传统应用** | 通用章节 |
| **AI 增强应用** | 通用章节 + AI 功能点表 + 提示词（简化版）+ 数据集 + AI 降级 |
| **AI Agent 应用** | 通用章节 + Agent 旅程 + Agent 故事 + 提示词（完整版）+ 数据集 + AI 降级 |

**三档需求规模自动适配：**

| 档位 | 触发条件 |
|------|---------|
| **快速模式** | 小功能迭代、单个需求点 |
| **标准模式** | MVP / 中等复杂度产品（默认） |
| **完整模式** | 正式产品 / 需交付给团队 |

**分两阶段交付：**
1. **概念版**（方向对齐）→ 先确认方向再展开细节
2. **落地版**（完整细节）→ 可直接交给 AI 或开发者实现

**自动补全小白盲区：**
- 交互细节（操作反馈、空状态、失败引导）
- 状态机（每个 UI 元素的所有状态）
- 数据规范（字段类型、长度、校验规则）
- 双受众文案规范（开发侧 vs 终端用户侧）
- 关键页面 ASCII 线框图（7 种页面类型模板）
- 用户故事（标准格式 + 验收标准）
- 异常处理与降级策略
- 测试标准与迭代规划

## 安装

把下面这条命令发给你的 AI 工具（Cursor、Windsurf、Claude Code、Codex 等），它会自动完成安装：

```
请将 https://github.com/RegenLau/regen-prd-writer.git 克隆到项目的 .claude/skills/ 目录下
```

或者手动执行：

```bash
cd /path/to/your/project/.claude/skills
git clone https://github.com/RegenLau/regen-prd-writer.git
```

安装后，提到「PRD」「需求文档」「我想做个产品」等关键词时，skill 会自动激活。

## 使用方法

直接用自然语言描述你的想法即可触发本 skill：

> "我想做一个帮自由职业者记录工作时间的工具"

> "我想做一个智能客服助手，能自动回答问题和处理退款"

> "帮我检查一下这份需求文档有没有问题"

> "帮我在现有文档里加一个通知功能的需求"

**触发关键词：**
需求文档、PRD、产品需求、功能文档、我想做一个产品/App/工具、帮我整理需求、帮我补充需求

## 工作流程

```
用户描述想法
     ↓
三视角诊断（用户需求 / 商业可行 / 技术落地）
     ↓
自动识别：产品类型（传统 / AI增强 / AI Agent）+ 需求规模（快速 / 标准 / 完整）
     ↓
输出【概念版】→ 用户确认方向（快速模式跳过此步）
     ↓
加载对应章节模板 → 输出【落地版】→ 可直接用于开发
  ├── 传统应用 → traditional-prd-chapters.md
  └── AI 应用 → ai-agent-prd-chapters.md（AI增强跳过 Agent 旅程/故事）
```
