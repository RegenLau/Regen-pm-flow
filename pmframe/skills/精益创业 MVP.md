# Lean Startup MVP

Validate a product idea by building the smallest possible version, measuring user behavior, and learning from the data. Use when you need to test a new product concept with minimal investment.

## Steps

1. State your **leap-of-faith assumption** — the riskiest belief that must be true.
2. Define **success metrics** — what measurable outcome proves the assumption.
3. Build the **MVP** — the smallest thing that tests the assumption (landing page, concierge, single feature).
4. **Measure** — deploy to real users and collect quantitative + qualitative data.
5. **Learn** — analyze results; decide to persevere, pivot, or kill.

## Output Format

A one-page MVP spec containing: the core assumption, success/failure criteria with specific thresholds, MVP scope description, measurement plan, and a decision framework for next steps.
