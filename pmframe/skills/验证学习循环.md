# Validated Learning Loop

Structure product discovery as a rapid cycle of forming hypotheses, running experiments, collecting data, and extracting actionable insights. Use when you need to learn quickly about users, market, or product viability.

## Steps

1. **Hypothesis** — state a falsifiable belief: "We believe [user segment] will [behavior] because [reason]."
2. **Experiment** — design the cheapest, fastest test to prove or disprove the hypothesis.
3. **Data** — collect quantitative metrics and qualitative observations.
4. **Insight** — analyze results; state what you learned and its confidence level.
5. **Decide** — persevere (double down), pivot (change direction), or kill (stop).
6. **Repeat** — feed the insight into the next hypothesis.

## Output Format

A learning card per cycle containing: hypothesis statement, experiment description, raw data summary, key insight, decision taken, and the next hypothesis queued.
