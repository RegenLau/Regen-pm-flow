# Problem-Solution Fit

Confirm that the problem is real, that users care about it, and that your solution addresses it effectively. Use before investing in full product development to avoid building something nobody wants.

## Steps

1. **Problem validation** — interview 10+ target users to confirm the problem exists, is frequent, and is painful.
2. **Current alternatives** — document how users solve the problem today and where those solutions fall short.
3. **Solution demo** — present your proposed solution (prototype, mockup, or description) to the same users.
4. **Fit assessment** — measure user reactions: willingness to switch, willingness to pay, excitement level.
5. **Iterate or proceed** — if fit is weak, refine the solution and retest; if fit is strong, move to MVP.

## Output Format

A fit report with: problem evidence (user quotes, frequency data), current alternatives analysis, solution feedback summary, a fit score (strong/weak/no fit), and recommended next steps.
