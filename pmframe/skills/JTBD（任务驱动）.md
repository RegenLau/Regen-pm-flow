# Jobs to Be Done

Identify the functional, emotional, and social "job" a user hires a product to accomplish, shifting focus from features to outcomes.

## Steps

1. **Identify the job** — Frame what the user is trying to achieve in a situation-motivation-outcome format: "When I [situation], I want to [motivation], so I can [outcome]."
2. **Map the job steps** — Break the job into sequential stages the user goes through to get it done.
3. **Find current solutions** — Document how users currently accomplish the job, including workarounds and competing products.
4. **Uncover struggling moments** — Identify where current solutions fail, frustrate, or underperform.
5. **Define success criteria** — Determine how users measure whether the job is done well.

## Output Format

A job statement in the standard format, a step-by-step job map, a list of competing solutions with their shortcomings, and prioritized underserved needs.
