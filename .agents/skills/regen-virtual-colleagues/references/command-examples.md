# 命令与触发示例

本文给出 `regen-virtual-colleagues` 在 V1 中的常见用户说法，帮助判断是项目级命令、话题级命令，还是圆桌/建议模式。

## 一、项目级命令

适用于长期团队配置。

### 启用 / 查看 / 调整

- “给这个项目绑定虚拟同事”
- “启用虚拟同事”
- “这个项目配几个虚拟同事”
- “查看当前项目虚拟团队”
- “把项目切成高频陪跑模式”
- “把项目切回关键节点模式”

### 增删成员

- “项目团队加一个前端同事”
- “给项目加一个增长同事”
- “把 Steve Jobs 加进项目团队”
- “项目团队删掉这个增长同事”

### 预期处理

- 作用域：项目级
- 结果：更新 `.regen/virtual-teams/project-team.json` 与 `registry.json`

---

## 二、话题级命令

适用于围绕某个议题临时组队。

### 新建或覆盖团队

- “这个话题单独配个团队”
- “针对这个问题组个团队”
- “这个议题不要沿用项目团队”
- “这个话题只听产品和技术的”
- “这个话题再加一个增长同事”

### 预期处理

- 作用域：话题级
- 结果：更新 `.regen/topics/<topic-id>/virtual-team.json`

---

## 三、日常建议命令

适用于让 1-3 位同事对当前问题发表建议。

- “让产品和前端同事先聊一下”
- “让技术同事看看这个方案”
- “问问虚拟同事这个想法”
- “@SteveJobs 你怎么看这个 MVP”
- “让这几个同事都给一句建议”

### 预期处理

- 先识别当前生效团队
- 再路由最相关的 1-3 位同事发言
- 输出摘要并落盘意见工件

---

## 四、圆桌命令

适用于多角色碰撞讨论。

- “开个圆桌”
- “围绕这个问题开会”
- “让他们一起讨论这个话题”
- “这个议题开个虚拟圆桌”
- “让团队辩一辩这件事”

### 预期处理

- 使用当前生效团队
- 主持人组织讨论
- 输出圆桌纪要
- 落盘到 `roundtables/`

---

## 五、与主 Skill 联动时的隐式触发

这些不是用户直接说给 `regen-virtual-colleagues` 的命令，而是其他主 skill 在关键节点可读取的触发点。

### `regen-competitor-deconstructor`

- 进入竞品分析前
- 用户准备确认竞品名单前

### `regen-material2proto`

- 进入原型生成前
- 结构化摘要准备确认前
- 原型范围准备定稿前

### `regen-proto2prd`

- 进入 PRD 生成前
- PRD 初稿准备交付前

### `regen-review-board`

- 正式评审前
- 用户准备采纳评审结论前

---

## 六、推荐回复格式示例

### 绑定团队时

```md
当前作用域：项目级
识别行业：AI工具
当前阶段：原型前

推荐候选同事：
1. Steve Jobs — 产品
2. Dan Abramov — 前端
3. Reid Hoffman — 增长
4. John Carmack — 工程
5. Julie Zhuo — 设计

待确认：
1. 你想绑定哪些同事？
2. 是否切到高频陪跑模式？
```

### 日常建议时

```md
当前问题：MVP 是否包含语音面试
触发来源：before_material2proto

本次发言同事：
- Steve Jobs
- Dan Abramov

汇总结论：
首版先验证文本主流程，语音能力延后。
```

### 圆桌时

```md
圆桌议题：MVP范围是否包含语音面试
参会同事：Steve Jobs / Dan Abramov / Reid Hoffman

当前建议结论：
先做文本主流程，把语音能力留到二期。

推荐下一步：
回到 regen-material2proto 调整页面范围。
```
