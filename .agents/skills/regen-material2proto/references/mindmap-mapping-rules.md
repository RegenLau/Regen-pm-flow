# XMind / 脑图 Markdown 到页面结构的映射规则

当输入材料中包含 XMind 导出的 Markdown 格式脑图时，使用以下规则将其映射为原型页面结构。

## 层级映射

| 脑图层级 | Markdown 标记 | 映射到原型结构                     |
|----------|---------------|-------------------------------------|
| 顶层主题 | `#`           | 产品名称 → 页面标题、侧边栏 Logo    |
| 一级模块 | `##`          | 功能模块 → 侧边栏的一级菜单项      |
| 二级功能 | `###`         | 页面/子功能 → 具体页面或标签页      |
| 列表项   | `- ` 或数字序号| 细节元素 → 字段、按钮、规则等       |

## 术语信号映射

脑图中的分支命名通常有固定含义，按以下规则映射到原型元素：

### 页面级映射

| 脑图分支名          | 原型页面类型               |
|---------------------|----------------------------|
| `列表页` / `列表`   | 带搜索、筛选、表格的列表页面 |
| `详情页` / `详情`   | 只读的信息展示页面          |
| `新建页面` / `新建`  | 包含表单的新建页面          |
| `编辑页面` / `编辑`  | 预填数据的编辑页面          |
| `设置` / `配置`     | 系统设置或配置页面          |
| `仪表盘` / `总览`   | 带图表和数据卡片的概览页面  |

### 组件级映射

| 脑图分支名            | 原型组件                               |
|-----------------------|-----------------------------------------|
| `说明`                | 页面顶部的说明文字或 tooltip             |
| `按钮`                | 页面操作区的按钮组                      |
| `列表展示` / `字段`   | 表格的列定义                            |
| `搜索条件`            | 搜索栏的筛选控件                        |
| `操作`                | 表格行内的操作按钮（编辑/删除/查看等）  |
| `文案` / `弹窗提示`   | 弹窗中的提示文案                        |
| `复选框`              | 多选控件                                |
| `switch开关`          | 开关切换组件                            |
| `状态`                | 状态标签（用不同颜色区分）              |

### 交互级映射

| 脑图分支名            | 原型交互行为                           |
|-----------------------|-----------------------------------------|
| `弹窗` / `抽屉`       | 点击后打开的浮层容器                    |
| `确认框`              | 二次确认弹窗（删除、状态变更等）        |
| `校验` / `校验规则`   | 表单验证逻辑（必填、格式、去重等）      |
| `下拉选择`            | Select 选择器组件                       |
| `级联选择`            | 多级联动选择器                          |
| `日期选择`            | DatePicker 日期选择组件                 |
| `文件上传`            | Upload 上传组件                         |

## 结构归位规则

1. **多页面模块**：如果同一模块下包含"列表页"、"新建页面"、"详情页"等多个分支，在原型中分别作为独立视图实现，通过导航串联

2. **弹窗/抽屉归属**：如果"按钮"或"操作"下包含"弹窗"或"抽屉"分支，该弹窗属于触发它的页面，在原型中作为该页面的浮层实现

3. **字段归属**：列表项（`- `开头）描述的字段，根据其父级分支归入对应的表格列、表单字段或搜索条件中

4. **示例数据**：如果脑图中出现示例值（如编码规则 `MDM-2024-001`），在原型中使用这些示例作为模拟数据

## 完整性要求

- 严格按照脑图中的出现顺序展开
- 保留所有节点和文本，不合并、不省略
- 保留原有中英文术语和缩写
- 字段名、按钮名在原型中应与脑图中完全一致
